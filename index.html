PROJECT AND REQUIREMENT PLANNING- PART 2

Project Concept



High Level Vision:

To meet business requirements of customers by providing effective and efficient software applications, maintaining high customer satisfaction and adapting to new software development technologies.

Rationale:

This software development project is designed to increase the user’s engagement as well as improving the quality of the current web application. The idea of the software project is to increase the customer satisfaction for the users of the system. In addition, the bottom line of the proposal software project is to enhance and offer top tier software quality solutions that can increase revenue for the university. Once the online web application is well built, customer experience will be better and in turn can bring the business to new heights.

Stakeholders:

The stakeholders of a project typically are people or groups affected by a software development project. The stakeholder of the software project includes the project team which consist of three members namely Jade Robinson, Michael Jones and Andre Cameron.Other stakeholders include the project sponsor, customers and end users.

Starting Assumptions and Constraints:

Starting Assumptions

1. Project Scope – the scope of what plan to deliver.

  • The project scope will remain the same once the stakeholders sign off on the scope statement.

2. Project Schedule: tasks, durations and dependencies needed to finish the project.

  • Materials will arrive on time and adhere to the project schedule.

3. Budget of the software project – estimated price of the project

  • Project costs will remain the same as initially budgeted expenditure.
  • Training will be conducted internally with no additional training expenses incurred.

4. Resources:

  • During the time they consent to tests, end users would be available to test.
  • Training rooms will be provided at the training center where necessary.

5. Our project team members have all the required skills to complete the project within time and budget.

Constraints

The constraints of this software development project will consist of the following:
  1. Time: There will be deadlines to be met and these will be given by the project sponsor. The project should be completed in one semester (13 weeks).
  2. Cost: The software project should be completed within budget that the project sponsor will propose. The project should be completed in a $10,000 USD budget.
  3. Scope: The scope describes what needs to be completed. An upgraded UtechOnline platform needs to be done.
  4. Quality: The quality constraint of the project will affect the cost of the project. If there is a strict quality requirement of the project, then it will directly affect the cost to complete the project. The quality of the project depends on the project sponsor’s budget.
  5. Risk: The risk constraint can be positive or negative. The positive risks must be enhanced, and the negative risk should be reduced in the project.
  6. Customer Satisfaction: This is a crucial constraint for the project because the customers are the source of income and reaching the bottom line of the business.
  7. Resources: The project team are not the only resources in the project but also the tools, equipment that are used during the duration of the project.

Roles and Timeline

Work Plan

Objective for Project:
  • To make an overall improvement of Utech’s online platform for students to interact with their teachers.
  • Implement an easy to find and use search bar by which students can access their course information quickly and efficiently.
  • Implement a system where modules that were done from previous semesters are still present on the dashboard even in the future semesters and students should only see modules that they are currently doing as it would be the focus for them.
  • To deliver an effective and efficient software platform within time and budget.
  • Improve the user interface to improve the useability on the site and make it more appealing to the eye to new and existing users of the site, hence making the user experience an overall better one.
